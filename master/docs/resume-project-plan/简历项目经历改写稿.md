## 项目经历

### 园区用能数据预测与异常预警平台

项目介绍：面向园区运营场景设计并实现用能数据预测与异常预警平台，基于历史用能、时间周期、温度、节假日等多维数据完成数据清洗、特征构建、趋势预测、异常波动识别和预警输出。系统支持预测任务调度、模型服务调用、预测结果入库、异常预警管理和可视化看板展示，用于辅助运营人员发现用能峰谷波动和异常风险。

技术栈：Java、Spring Boot、Spring MVC、Spring Data JPA、MySQL、Redis、Quartz、Python、Pandas、NumPy、scikit-learn、FastAPI、ECharts、Maven

项目亮点：

1. 使用 Pandas/NumPy 对小时级用能数据进行清洗和时间对齐，构建小时、星期、节假日、温度、滞后值、滑动均值、滑动标准差等时序特征，提升模型对周期性用能规律的刻画能力。
2. 实现历史同期基线模型与机器学习模型的效果对比，使用 MAE、RMSE、MAPE 等指标评估预测结果，并基于预测残差分析模型误差来源。
3. 基于预测残差阈值和 Isolation Forest 识别异常波动，生成低、中、高不同等级预警记录，支持异常原因、实际值、预测值和残差结果追踪。
4. 使用 FastAPI 封装 Python 预测与异常检测服务，Java 后端通过 REST API 调用模型接口，并将预测结果和预警记录写入 MySQL。
5. 使用 Quartz 定时触发数据更新、趋势预测和预警同步任务，结合 ECharts 展示用能趋势、预测曲线、异常点和模型评估指标。

可量化表述候选：

- 完成小时级用能数据预测、残差分析和异常预警闭环。
- 使用 MAE/RMSE/MAPE 对模型效果进行评估，并与历史同期基线模型进行对比。
- 理论验证中，模型 MAE 相比基线降低约 49.57%，MAPE 约 5.19%。

注意：如果没有将模拟数据改成真实或公开数据，面试时可以说“先用模拟园区用能数据验证完整链路”，不要说成真实电力交易数据。

### 校园学生行为分析与风险预警系统

项目介绍：面向校园管理场景设计并实现学生行为分析与风险预警系统，基于门禁日志、借阅日志、网络日志和用户基础信息等多源数据，完成数据清洗、用户维度特征融合、行为画像、聚类分析和风险预警。系统支持账号权限管理、任务调度、分析结果同步、预警处理和可视化看板展示。

技术栈：Java、Spring Boot、Spring MVC、Spring Data JPA、MySQL、Redis、Quartz、Python、Pandas、NumPy、scikit-learn、PyTorch、FastAPI、K-Means、Maven、HTML、CSS、JavaScript、ECharts

项目亮点：

1. 完成门禁、借阅、网络等 90 万+ 多源日志的数据清洗、聚合和用户维度融合，围绕 1000+ 用户构建日均在线时长、学习流量占比、图书馆访问次数、晚归次数、借阅次数、活跃天数等 18 项行为特征。
2. 将 Java 本地聚类调整为 Python 机器学习服务，通过 Pandas/NumPy 构建特征矩阵，使用 StandardScaler 完成特征标准化，并基于 scikit-learn K-Means 生成用户行为簇。
3. 引入 PyTorch AutoEncoder 作为行为特征表征增强模块，学习低维行为表示后再进行 K-Means 聚类，输出簇标签、簇中心、样本分配、Silhouette Score 和 Davies-Bouldin Score 等评估结果。
4. Java 后端通过 PythonClusterClient 调用 FastAPI 聚类服务，由 KMeansService 负责聚合用户特征、处理模型服务响应、写入用户画像结果，并在服务不可用时给出明确异常提示。
5. 结合规则评分与聚类结果计算综合风险分和分维度风险分，支持网络行为异常、作息波动、学习投入变化、借阅异常等预警类型。
6. 使用 Quartz 实现分析快照、画像更新和预警同步任务调度，通过 Spring Boot、MySQL、Redis 实现权限管理、数据查询、预警处理、日志导出和看板展示。

可量化表述候选：

- 处理 1000+ 用户、90 万+ 多源行为日志。
- 构建 18 维用户行为特征，支持用户画像聚类和分维度风险预警。
- Python 聚类服务返回簇中心、样本分配和模型评估指标，Java 后端负责业务落地与看板展示。
